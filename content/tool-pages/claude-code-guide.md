# Claude Code：新手学习页

## 一句话解释

Claude Code 是由 Anthropic 官方推出的**命令行 AI 编程 Agent（智能体）**。它直接运行在你的电脑终端（命令行）中，能够“看懂”你整个项目的目录结构，自主查找文件、运行本地测试、自动修复开发报错，并能跨越多个文件同时进行修改。

---

## 适合谁

* **数字化先锋/技术项目对接人**：需要频繁测试运行各种小工具，排查脚本为什么跑不起来。
* **有一定基础的半专业开发者**：希望 AI 不仅能“提供修改代码的建议”，还能像一个助理一样直接在本地“把代码改好并运行通过”。
* **追求极速与高效率的极客**：不喜欢在不同的聊天网页和代码编辑器之间来回切换，希望在终端命令行里解决所有问题。

## 不适合谁

* **对命令行极度恐惧的小白**：如果你从未用过终端控制台（如 CMD、PowerShell、Terminal），看到一行行滚动代码就头疼，建议优先使用界面更直观的图形化工具 [Cursor](./cursor-beginner-guide.md) 或 [Trae](./trae-guide.md)。

---

## 它能帮你做什么

| 能力 | 新手理解 | 示例任务 |
| :--- | :--- | :--- |
| **自主扫描与诊断** | 能够自行阅读并分析整个文件夹的文件，不需要你手动一段段把代码复制给它。 | 输入：“阅读这个项目的 index.html 和 build.ps1，帮我画出它们的逻辑调用关系。” |
| **跨文件协同修改** | 能够根据一个指令，同时修改 HTML、CSS 和配置文件，确保改动前后一致。 | 输入：“在导航栏加个新按钮，同步编写它的弹窗样式，并在配置表里注册它。” |
| **自动排错并验证** | 它可以自己在控制台运行本地测试脚本，如果报错，它会根据报错日志自己修改代码直到跑通。 | 输入：“运行 `./scripts/build.ps1`，如果有编译报错，分析原因并帮我修好它。” |

## 使用前准备

| 准备项 | 是否必须 | 说明 |
| :--- | :--- | :--- |
| **Node.js 环境** | 是 | 需要在电脑上安装 Node.js（v18.0 或更高版本）。可在终端运行 `node -v` 验证是否成功。 |
| **Git 版本控制** | 是 | **极其重要！** 你的项目目录必须经过 Git 初始化（有 `.git` 文件夹）。因为 Claude Code 修改代码是直接覆盖本地文件的，没有 Git 你将无法撤销 AI 的改动。 |
| **Anthropic API 账号** | 是 | 需要有 Anthropic Console 开发者账号（有可用额度余额），或拥有 Claude Pro 订阅会员身份以进行 Web 登录授权绑定。 |

## 第一次使用流程

### 步骤 1：检查你的 Node.js 环境
1. 按下快捷键 `Win + R`（Mac 用户打开 Terminal），输入 `powershell` 回车。
2. 在弹出的窗口中输入：`node -v`，如果正确吐出如 `v20.11.0` 等版本号，说明准备就绪。

### 步骤 2：在全局安装 Claude Code
1. 在你的 PowerShell 终端窗口中，直接输入以下全局安装命令并回车：
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```
2. 等待绿色滚动条进度跑完，控制台重新出现输入提示符。

### 步骤 3：进入项目并启动 Claude Code
1. 在终端切换到你要修改的项目的文件夹下（例如：`cd d:\my-project`）。
2. 输入启动命令并回车：
   ```bash
   claude
   ```
3. 首次启动时，终端会给出一个 8 位数字验证码，并自动在浏览器中弹出一个 Auth（授权）网页。
4. 登录你的 Anthropic 账号，粘贴验证码并授权通过。授权成功后，你的控制台会变成 Claude 的专属暗色对话界面。

### 步骤 4：直接下达指令
1. 比如直接对它说：“帮我看一下当前项目下有哪些 Markdown 教程，把它们的标题都列出来。”
2. 观察 AI 自行使用 `find_files` 等工具进行搜索并给出完美的中文解答。
3. 退出时，输入 `/exit` 即可安全返回普通控制台。

## 推荐新手任务

### 任务：自动项目诊断
* **任务目标**：让 AI 主动理清当前老旧项目的文件关系，找出可以优化的地方。
* **操作方式**：在启动 `claude` 后的对话框中输入：
  ```text
  请先不要修改任何文件。帮我通读这个项目的核心目录，用大白话列出：
  1. 这个项目最核心的文件是哪几个？它们是如何分工协作的？
  2. 针对当前项目，你有哪些代码优化或安全性排查的建议？
  ```
* **预期效果**：AI 会自主运行文件搜索和读取工具，最后梳理出一份条理清晰的中文诊断报告。

## 常见错误与排错

| 报错现象/常见错误 | 可能原因 | 解决方法 |
| :--- | :--- | :--- |
| **`npm ERR! code EACCES`** | 在全局安装包时，当前终端没有足够的管理员/写入权限。 | **Windows**：右键点击 PowerShell 图标，选择“以管理员身份运行”，重新运行安装命令。<br>**Mac**：在命令前加上 `sudo`（如 `sudo npm install -g ...`）。 |
| **修改后代码出错，想全部撤销** | AI 在自主排错时逻辑走偏，越改越乱，导致项目无法编译。 | 在你的普通命令行终端（不要在 Claude 内部）运行：<br>`git reset --hard HEAD`<br>即可安全将所有文件回退到 AI 动手前的状态。 |
| **`mcpTool` 权限阻断** | 频繁运行外部命令时系统频繁弹出安全警告。 | 可以在启动时使用 `--approved-tools` 参数，或仅在可信任的本地工作区内运行 Claude Code。 |

## 推荐 Prompt

### 让 Claude Code 帮你进行自动代码重构与跑通测试
```text
请帮我查找项目里所有的 .md 文件，将它们的第一行标题格式规范化（比如去除前后的多余空格，并确保是 # 一级标题开头）。
修改后，请在本地自动运行编译命令 `./scripts/build.ps1`，确保编译结果没有报错后再告诉我。
```

## 和同类工具的区别

* **vs. Cursor / Trae (图形化编辑器)**：Cursor 像一个全功能的 IDE 编辑器，你一边看着代码，一边让 AI 在右侧帮你写；而 **Claude Code** 完全是在终端里的 Agent，适合你懒得动手、直接下达宏观指令（如：“帮我把所有过期的依赖包都升级，并测通”）的自动化运维场景。
* **vs. 网页版 Claude.ai**：网页版只能读取你手动上传的文件，且无法在你的电脑上运行命令；而 **Claude Code** 拥有执行命令行、搜索本地文件、查看 Git 提交记录等真实本地开发能力。
