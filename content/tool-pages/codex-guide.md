# OpenAI Codex 学习指南

## 一句话解释

Codex 是由 OpenAI 研发的底层代码理解与生成模型（驱动了早期的 GitHub Copilot）。它能将自然语言直接翻译成代码，不直接提供图形界面，而是作为 API 接口供其他软件或插件调用。

## 适合谁

* 数字化团队中需要编写自定义脚本、批量调用 AI 接口处理 Excel 数据或进行数据库清洗的工程师。
* 想要自己开发定制化 AI 插件、内网 AI 助手的技术极客。

## 首次使用流程

### 第一步：获取 API Key
登录 OpenAI 平台，创建一个 API 密钥并妥善保存。

### 第二步：在 Playground 测试
打开 OpenAI Playground，选择 Code 模式，直接用自然语言编写注释，看它如何实时自动吐出 Python 或 SQL 代码。

### 第三步：代码调用
在本地 Python 脚本中引入 `openai` 库，通过发送 prompt 调用 Codex 接口进行批量文本结构化处理。
```python
import openai
# 示例：通过 API 请求进行代码解析
```

## 常见错误

* **误当成独立编辑器**：Codex 不是一个像 VS Code 那样双击打开就能写字的代码编辑器，它是一个后台模型接口，需要配合 API 或编辑器插件使用。
* **直接执行返回代码**：在编写自动化脚本时，严禁让程序直接 `eval()` 执行 Codex 返回的命令，防范 AI 产生幻觉生成了破坏性文件删除指令。

---

## 防泄密安全提醒

* **API 密钥硬编码**：千万不要把 API Key 直接写在 Python 代码的明文里（如 `api_key = "sk-..."`）并上传到 GitHub。**必须使用环境变量 `process.env` 或 `.env` 文件进行隔离**。
