# Cursor 新手学习页

## 一句话解释

Cursor 是一个带 AI 能力的代码编辑器。你可以把它理解成“会帮你读代码、改代码、解释项目的开发工具”，适合想用 AI 学编程、改网页、理解 GitHub 项目的新手。

## 适合谁

- 想用 AI 辅助写代码，但不知道从哪里开始的人。
- 想打开 GitHub 项目，让 AI 解释项目结构的人。
- 想修改网页文案、按钮、样式、简单功能的人。
- 已经会一点点电脑操作，但不熟悉命令行和项目结构的人。
- 想学习 AI 编程工作流，而不是只和聊天机器人对话的人。

## 不适合谁

- 完全不想接触文件、项目、代码目录的人。
- 希望 AI 一次性自动做完所有开发任务、自己不检查结果的人。
- 需要处理大型生产项目，但还不了解版本管理和备份的人。
- 不愿意阅读 AI 修改内容，也不愿意测试结果的人。

## 它能帮你做什么

| 能力 | 新手理解 | 示例任务 |
|---|---|---|
| 解释项目结构 | 让 AI 告诉你每个目录大概做什么 | “这个项目的首页文件在哪里？” |
| 修改代码 | 根据你的要求改某个文件 | “把按钮文案改成‘开始学习’。” |
| 生成代码 | 根据目标生成页面、组件或函数 | “帮我做一个工具卡片组件。” |
| 修复报错 | 根据报错信息分析原因 | “为什么运行后显示 module not found？” |
| 重构代码 | 把代码整理得更清晰 | “把重复的卡片数据抽成数组。” |
| 写说明文档 | 生成 README、注释或使用说明 | “给这个项目写一段安装说明。” |

## 使用前准备

| 准备项 | 是否必须 | 说明 |
|---|---|---|
| Cursor 软件 | 必须 | 需要从官方渠道下载安装。具体下载地址和版本以官方页面为准。 |
| 一个项目文件夹 | 建议 | 可以是你自己创建的文件夹，也可以是从 GitHub 下载的项目。 |
| GitHub 账号 | 可选 | 如果你只打开本地文件，不一定需要账号；如果要 Clone 或提交代码，建议准备。 |
| Node.js / Python 等运行环境 | 视项目而定 | 如果项目需要运行，通常 README 会说明需要什么环境。 |
| API Key | 视功能而定 | 某些模型或服务可能需要 API Key。API Key 是证明你有权限调用服务的一串密钥。 |

## 第一次使用流程

### 步骤 1：安装并打开 Cursor

从官方渠道下载安装 Cursor。安装完成后打开软件，如果提示登录或选择模型，按页面提示完成基础设置。具体登录方式、可用模型和收费规则可能变化，第一次使用时以 Cursor 当前页面显示为准。

成功后你应该看到类似代码编辑器的界面，通常会有文件列表、编辑区域和 AI 对话区域。

### 步骤 2：打开一个项目文件夹

选择一个你想学习或修改的文件夹。如果你还没有项目，可以先创建一个空文件夹，或者从 GitHub 下载一个简单项目。

如果你看到左侧出现文件列表，说明项目已经打开。文件列表通常会包含 `README.md`、`package.json`、`src`、`public` 等文件或目录。

这些词可以这样理解：

- `README.md`：项目说明书，通常写着项目用途、安装方式和运行命令。
- `package.json`：Node.js 项目的配置文件，通常写着依赖和可运行命令。
- `src`：主要源代码目录，很多功能代码会放在这里。
- `public`：静态资源目录，常见图片、图标等可能放在这里。

### 步骤 3：先让 AI 解释项目，不要急着修改

第一次打开项目时，不建议马上让 AI 改代码。先让 AI 解释项目结构，确认你知道它要改哪里。

可以复制这个 Prompt：

```text
请你先阅读当前项目结构，然后用新手能理解的语言解释：
1. 这个项目是做什么的
2. 主要目录分别有什么作用
3. 我应该从哪个文件开始看
4. 如果我要修改首页，应该看哪些文件
5. 哪些文件不建议新手随便改
```

成功后，你应该得到一段项目说明，并看到 AI 提到具体文件或目录。

### 步骤 4：给 AI 一个小而明确的任务

新手第一次不要让 AI “重构整个项目”或“做一个完整系统”。任务越小，越容易检查。

适合第一次尝试的任务：

- 修改首页标题。
- 修改按钮文案。
- 增加一个简单卡片。
- 解释某个报错。
- 给 README 补一段运行说明。

示例 Prompt：

```text
请帮我把首页主标题改成“Agent 学习地图”。
请先告诉我你准备修改哪个文件，等我确认后再修改。
```

这个 Prompt 的重点是“先说明，再修改”。这样可以减少 AI 改错文件的风险。

### 步骤 5：检查 AI 的修改结果

AI 修改代码后，你需要检查两件事：

- 文件内容是否符合你的要求。
- 项目运行后页面是否正常。

如果项目有运行命令，通常 README 或 `package.json` 中会写。常见命令可能是：

```text
npm install
npm run dev
```

`npm install` 的意思是安装项目需要的工具包；`npm run dev` 的意思是以开发模式运行项目。不同项目命令可能不同，不要在不了解含义时随意运行复杂脚本。

### 步骤 6：保存或回退修改

如果修改结果正确，可以保存文件。如果结果不对，先不要继续追加更多要求，应该让 AI 解释它改了什么。

可以问：

```text
请总结你刚才修改了哪些文件、每个文件改了什么、如果我要回退应该怎么做。
```

如果你正在学习 Git，建议每次完成一个小修改后提交一次。这样出错时更容易回退。

## 推荐新手任务

### 任务 1：让 Cursor 解释项目结构

目标：知道项目大概由哪些部分组成。

Prompt：

```text
请用新手能理解的语言解释当前项目结构，并告诉我如果只想修改首页，应该先看哪些文件。
```

完成标准：AI 能指出 2-5 个关键目录或文件，并说明每个文件的用途。

### 任务 2：修改一个页面文案

目标：完成第一次低风险代码修改。

Prompt：

```text
请把页面上的“Welcome”改成“欢迎来到 Agent 学习地图”。
请先说明你会修改哪个文件，等我确认后再改。
```

完成标准：页面文案被修改，项目没有新增报错。

### 任务 3：让 Cursor 解释一个报错

目标：学会把报错交给 AI 分析，而不是只复制最后一行。

Prompt：

```text
请根据下面的完整报错分析原因。先不要修改代码。
请按这个格式回答：
1. 报错是什么意思
2. 最可能的原因
3. 我应该先检查哪些文件
4. 如果要修改，建议怎么改
5. 修改前需要我确认什么

报错信息：
[粘贴完整报错]
```

完成标准：你知道下一步该检查哪个文件，而不是只得到一句“重新安装依赖”。

## 常见错误

| 问题 | 可能原因 | 解决方法 |
|---|---|---|
| AI 改了很多文件 | 任务范围太大，或者没有要求先说明修改计划 | 下次使用“先说明修改文件，等我确认后再改” |
| 项目运行失败 | 没有安装依赖、Node/Python 版本不匹配、命令用错 | 先看 README，再把完整报错发给 AI 分析 |
| AI 解释太专业 | Prompt 没有说明“用新手能理解的语言” | 加上“请解释术语，不要默认我懂命令行” |
| 不知道改哪里 | 没有先让 AI 分析项目结构 | 先使用项目结构解释 Prompt |
| API Key 泄露风险 | 把密钥写进代码或公开上传到 GitHub | 使用 `.env` 文件保存，公开前检查是否包含密钥 |

## 推荐 Prompt

### 项目结构解释

```text
请你先阅读当前项目结构，然后用新手能理解的语言解释：
1. 这个项目是做什么的
2. 主要目录分别有什么作用
3. 我应该从哪个文件开始看
4. 如果我要修改首页，应该看哪些文件
5. 哪些文件不建议新手随便改
```

### 小范围修改

```text
请帮我完成这个小修改：[写清楚你的目标]。
要求：
1. 先告诉我你准备修改哪些文件
2. 不要一次性重构整个项目
3. 修改后说明每个文件改了什么
4. 如果有风险，请提前告诉我
```

### 报错分析

```text
请先不要修改代码。请根据报错信息判断：
1. 报错含义
2. 最可能原因
3. 需要检查的文件
4. 建议修改步骤
5. 修改前需要我确认的问题
```

## 和同类工具的区别

| 工具 | 更适合什么 | 新手注意点 |
|---|---|---|
| Cursor | 在编辑器里读代码、改代码、完成项目内任务 | 要学会控制修改范围，不要盲目接受所有改动 |
| Claude Code | 更偏命令行和项目级 Agent 工作流 | 对命令行和项目结构要求更高 |
| Codex | 适合把自然语言任务转成代码修改或开发方案 | 具体能力和入口以当前产品形态为准 |
| 普通聊天机器人 | 解释概念、写代码片段、生成思路 | 不一定能直接读取你的本地项目文件 |

## 学完后下一步

建议按这个顺序继续学习：

1. 从 GitHub 下载一个简单项目。
2. 用 Cursor 打开项目。
3. 让 Cursor 解释项目结构。
4. 修改一个页面文案或按钮。
5. 学会用 Git 保存修改。
6. 再尝试让 AI 修复一个简单报错。

如果你能完成这些任务，就已经从“只会问 AI 问题”进入到“能用 AI 处理一个小项目”的阶段。
