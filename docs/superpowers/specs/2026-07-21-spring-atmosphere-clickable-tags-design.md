# 春日雾感背景与可点击课程标签设计说明

## 目标

在不改变现有学习内容、章节顺序、阅读器、进度记录、搜索和构建流程的前提下，改善首页的整体氛围，并让课程卡片中的标签成为明确、可访问、可点击的学习入口。

## 视觉方向

页面采用原创的“春日雾感草地”视觉，不直接使用参考图，也不复制其文字、标识或专属素材。背景图以深浅绿色草地、柔焦紫白花影、薄雾和轻微纸张颗粒为核心元素，为标题与卡片预留低细节区域。

- 首屏展示背景的主要构图，并使用深绿色遮罩维持标题对比度。
- 页面主体延续柔和的绿色氛围，但降低背景细节与对比度，避免影响阅读。
- 内容层使用暖白实色或高不透明度表面，不采用大面积透明玻璃效果。
- 现有黑白基础、字体层级、8px 卡片圆角、24px 大面板圆角和间距体系继续保留。
- 章节色块改为较克制的识别色，绿色、紫色、米色只承担分类与状态提示，不与背景争夺注意力。
- 用户启用“减少动态效果”时，所有背景漂移和视差均停止。

## 原创背景资产

新增一张宽幅 WebP/JPEG 背景资源，保存于 `prototype/assets/`。画面不得包含文字、Logo、人物或参考图中的特定构图。背景资源需提供 CSS 渐变兜底，资源加载失败时页面仍保持可读。

移动端使用同一资源的居中裁切，并减少装饰层；不额外加载多张大图，控制首屏负担。

## 首页结构

### 首屏

首屏保持现有标题、简介、操作按钮、进度卡和学习步骤。背景承担氛围，信息区使用深绿与暖白的高对比组合。原有纸片式装饰简化为少量课程路径标记，避免遮挡标题和按钮。

### 章节面板

章节继续按照当前业务顺序呈现。每个章节使用暖白内容面板加细边框，通过标题旁的色签区分类型。深色章节保留，但减少纯黑大块面积，卡片统一使用清晰的实色表面。

## 可点击课程标签

所有拥有 `data-key` 的课程卡都必须具备以下交互：

1. 整张卡片可用鼠标点击，打开现有阅读器中的对应内容。
2. 卡片可通过 Tab 聚焦，并可用 Enter 或空格打开。
3. 卡片内的课程类型或主题标签改为真实按钮式入口，例如“AI 海报”“Cursor”“SOP”。
4. 标签点击时打开该卡片对应的教程，并阻止事件冒泡，避免重复触发卡片点击。
5. 卡片底部增加明确的“开始学习 →”行动提示。
6. 标签与卡片均提供悬停、按下和 `:focus-visible` 状态，点击区域最低 44px。
7. 已完成课程仍使用现有完成状态，不改变进度数据结构。

标签只作为对应教程的快捷入口，不创建不存在的新文章，不改变 `content` 对象与 Markdown 来源的映射关系。

## 数据与事件流

页面加载后，脚本遍历所有包含 `data-key` 的课程卡：

- 为缺少语义的卡片补充 `role="button"`、`tabindex="0"` 和可读的 `aria-label`。
- 将卡片内的 `.kind` 与指定主题标记初始化为课程入口按钮。
- 标签入口从最近的卡片读取 `data-key`，调用现有 `openReader(key)`。
- 搜索继续读取标题、简介和标签文本；标签按钮化不会改变搜索结果。
- 阅读器关闭、完成状态和进度统计继续沿用现有逻辑。

若某张卡缺少 `data-key` 或对应内容不存在，脚本不生成入口，并保留现有的内容缺失提示，避免出现无响应按钮。

## 响应式与可访问性

- 960px 以下继续使用折叠导航。
- 768px 以下章节面板贴近屏幕边缘，背景装饰降低强度。
- 560px 以下首屏按钮和课程入口可换行，所有操作区域不小于 44px。
- 正文、标签、按钮与背景满足清晰对比；不把颜色作为唯一状态提示。
- 所有交互同时支持鼠标、触屏和键盘。

## 验证标准

- 构建脚本可以重新生成页面内容而不覆盖视觉和交互代码。
- 所有带 `data-key` 的课程卡都具有键盘语义与独立标签入口。
- 每个标签入口打开与所属卡片相同的阅读内容。
- 搜索、阅读器、完成状态、进度统计、移动导航和 UI/UX 三张课程卡正常工作。
- 背景资源缺失时 CSS 兜底仍可读。
- 运行项目现有构建、UI/UX 校验和 Git 差异检查均无错误。

## 非目标

- 不新增课程文章或修改课程正文。
- 不改变学习进度的本地存储格式。
- 不复制参考图片、品牌名称、文字或专属素材。
- 不引入框架、服务端或第三方运行时依赖。
