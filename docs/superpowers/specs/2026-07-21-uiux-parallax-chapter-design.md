# UI/UX 递进章节与视差首屏设计

## 目标

在不破坏现有学习地图阅读效率的前提下，新增一套可长期复习的 UI/UX 学习章节，并用轻盈的拼贴式视差首屏提升首页的视觉记忆点。

## 已确认方向

- 新章节采用三张递进课程卡片，每张可独立阅读。
- 视差效果应用于首页首屏与新增 UI/UX 章节；其他学习内容继续使用现有卡片布局。
- 视觉参考为用户提供的绿色渐变拼贴图：白色留白、纸张/便签/界面截图层叠、圆角卡片、轻微旋转、柔和阴影和清晰黑色标题。
- 动效参考 NameThatUI 的 parallax 定义：背景、中景、前景以不同速率随滚动位移，制造景深。

## 信息架构

新增顶部导航入口“UI/UX 实操”，并在现有“从想法到 PRD”章节之后、“部署与分享”之前插入新章节。课程顺序如下：

1. **Part A｜从 PRD 迈向 UI**：理解产品边界、生成低保真流程、建立页面资产台账、确定设计规则并完成交付验收。
2. **Part B｜Call Insight × Stitch 实操**：使用设计简报和低保真线框图生成、优化并交付高保真移动端页面；附完整 Stitch brief 与线框图。
3. **Part C｜UI/UX 工具地图**：按“从零生成、成品优化、交互打磨、设计规范沉淀”四类任务选择工具。

每张卡片打开现有站内阅读器。文章内提供明确的“查看原始 HTML 演示”或“查看高清线框图”入口，避免把大量展示内容硬塞进正文。

## 首页首屏

首屏改为一块横向“AI 学习工作台”拼贴画布：

- 左侧保留主标题、项目定位、搜索和学习进度入口。
- 中央使用三张错位叠放的课程预览纸片，分别代表“想法 → PRD”“PRD → UI”“UI → 可交付产品”。
- 周围加入便签标签、路线箭头、圆点和小型工具卡作为装饰，不使用与学习主题无关的人像素材。
- 采用明亮春绿色到近白色的纵向渐变背景，主文字保持黑色或深墨色，强调色沿用绿色并少量加入淡蓝、黄色。
- 保留现有功能入口，但把它们整合为首屏中的可点击纸片或浮动按钮。

## 新章节视觉

- 章节顶部是一条较短的拼贴式舞台，展示 Call Insight 线框图局部、Stitch 设计简报纸片和工具标签。
- 三张主课程卡按 01、02、03 编号，桌面端横向递进，移动端纵向排列。
- 卡片用不同的轻微旋转角度和重叠关系形成手作感；悬停或键盘聚焦时回正、上浮并显示“开始学习”。
- 卡片下方提供一条“最终成果路径”：PRD → 线框图 → 高保真 UI → 前端 Demo。

## 动效与实现边界

- 使用现有静态 HTML/CSS/JavaScript 架构，不引入动画框架或新的构建依赖。
- 视差只更新 `transform` 和少量 `opacity`，通过一次 `requestAnimationFrame` 合并滚动更新，避免滚动卡顿。
- 背景、中景、前景分别使用约 0.15、0.45、1.0 的视觉速度差；控制总位移，避免文字和按钮漂移到不可点击位置。
- 重要正文和操作按钮不做持续视差，确保阅读和交互稳定。
- `prefers-reduced-motion: reduce` 时完全关闭视差、悬停旋转和过渡动画。
- 窄屏下缩小位移幅度并减少装饰层，避免横向溢出；不使用 `background-attachment: fixed`。

## 内容与资源

- 新建 `content/uiux/` 保存三篇课程正文和 Part B 的完整设计简报。
- 将两份原始 HTML 演示复制到站点可访问的 UI/UX 资源目录。
- 将 Call Insight 低保真线框图复制为站点资源，并提供可放大/新窗口查看的链接与替代文本。
- 构建脚本增加对应内容键，生成后的阅读器继续使用当前内容嵌入方式。
- README 的内容目录与课程数量同步更新。

## 验收标准

1. 首页首屏能明显呈现用户参考图的绿色渐变、纸张拼贴和层叠感。
2. 滚动首屏或 UI/UX 章节时可观察到至少三层不同速度的位移，且无明显卡顿。
3. 顶部导航可直达 UI/UX 新章节，三张课程卡均能打开正确正文。
4. Part A、Part C 正文能打开对应原始 HTML 演示；Part B 能打开 Stitch brief 和高清线框图。
5. 现有教程、搜索、阅读器、学习完成状态和 Prompt 工具保持可用。
6. 桌面端与移动端不出现横向滚动条，文字和按钮不被装饰遮挡。
7. 系统启用“减少动态效果”后，页面切换为稳定的静态拼贴布局。
8. 内容构建成功，生成页面中的新增课程数据完整。

## 非目标

- 不重写现有站点为 React、Vue 或其他框架。
- 不把全部旧章节改造成视差叙事页。
- 不制作 Call Insight 产品本身的可运行 Demo。
- 不引入登录、云端进度同步或新的后端能力。
