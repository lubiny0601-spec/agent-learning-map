# Prompt 优化助手 (Prompt Optimizer Skill)

## 目标

此 Skill 用于引导 Agent 将初学者模糊、简单的自然语言指令，转化为结构清晰、指令明确的高质量结构化提示词（Prompt），供 AI 编程助手（如 Cursor、Claude Code、Antigravity 等）使用。

初学者往往喜欢发送类似“帮我做一个博客页面”或“修改一下样式”这种非常宽泛的指令，这极易导致 AI 产生学术幻觉、写出破损的代码，或是进行大范围不可控的修改。此 Skill 旨在约束 Agent 生成的 Prompt，使其强制 AI 遵循安全、渐进式且高度可验证的开发工作流。

在用户请求进行以下操作时调用此 Skill：
- 为特定开发任务生成 Prompt 模板。
- 优化他们准备发送给其他编程助手的原始 Prompt。
- 创建系统提示词或规则文件（如 `.cursorrules` 或 `AGENTS.md`）。

## 输入要求

尽可能多地收集以下上下文信息。如果某些细节缺失，请使用标准的行业假设（例如：前端默认使用 vanilla JS/CSS，脚本默认使用 Python），并在占位符中标记为 `[待填写]`：
- **原始需求**：用户想要实现什么功能（例如：“增加一个登录功能”）。
- **目标 AI 工具**：Cursor Composer、Claude Code、Antigravity 等。
- **技术栈**：HTML、CSS、React、Node.js、Python 等。
- **项目上下文**：是一个全新项目还是在一个现有的代码库上修改？
- **约束条件**：设计规范、性能要求或特定的库限制。

## 撰写原则（结构化 Prompt 框架）

此 Skill 生成的优化 Prompt 必须遵循专为软件工程适配的简化 **CO-STAR** 框架：

1. **Role & Context (角色与上下文)**：定义 AI 的专业角色（例如：“你是一个资深 React 开发专家”）并说明项目的背景。
2. **Objective (明确任务)**：将原始任务拆解为 3-5 个具体的、有先后顺序的步骤，避免含糊的目标。
3. **Tech Stack & Scope (技术栈与范围)**：列出明确使用的技术，以及**只允许**修改的文件范围。
4. **Defensive Rules (防御性指令 - 至关重要)**：加入严格的安全红线以防止 AI 破坏现有代码：
   - “严禁重写整个文件，只能进行精准的局部替换。”
   - “必须完整保留所有现有的代码注释、文档说明和无关样式，不得擅自清理。”
   - “在安装任何新依赖包或删除文件之前，必须先征得我的口头确认。”
   - “如果遇到报错，必须一步步分析原因，严禁盲目猜测解决方案。”
5. **Output Format (输出格式)**：告诉 AI 应该如何回复（例如：“请先列出修改的文件清单，然后解释具体改动，最后提供验证命令”）。

## 输出结构

除非用户另有要求，否则请使用以下结构：

```markdown
# Prompt 优化方案

这是为您优化后的结构化 Prompt。你可以直接复制框内的内容发送给您的 AI 编程助手。

## 💡 为什么这样优化
简要说明相比原 Prompt，这个版本做了哪些关键改进（如：限制了修改范围、增加了安全防护、明确了技术栈）。

## 📋 复制下方内容发送给 AI

```text
【角色定义】
你是一个优秀的 [技术栈，如：HTML/JS] 专家。请协助我完成以下开发任务。

【当前上下文】
- 项目类型：[如：静态网页原型]
- 目标文件范围：仅限修改 [文件路径/名称，如：prototype/index.html]，禁止擅自修改其他目录的文件。

【核心任务】
请逐步执行以下操作：
1. [步骤 1，如：在导航栏右侧增加一个“生成器”按钮]
2. [步骤 2，如：为该按钮绑定点击事件，弹出一个简单的模态窗口]
3. [步骤 3，如：在模态窗口中包含三个输入框和一个复制按钮]

【技术要求与约束】
- 使用 Vanilla CSS 进行样式设计，禁止引入第三方 CSS 框架（除非我明确要求）。
- 保持现有的设计美学（如 HSL 色彩配置、圆角过渡等），确保风格一致。
- 确保页面在移动端（响应式）的表现依然良好。

【🛡️ 防御性安全规则】
1. **禁止大范围重写**：只对需要修改的 HTML 代码块进行局部替换，保留所有未涉及的结构和注释。
2. **确认后再操作**：在修改任何文件之前，先向我口头确认你的修改方案和将要修改的文件路径。
3. **禁止擅自下载**：不要运行任何下载依赖或执行外部脚本的命令。

【输出要求】
修改完成后，请告诉我：
1. 你具体修改了哪些行（提供 Diff 或对比说明）。
2. 在浏览器中如何测试该功能是否正常工作。
```

---

## 🛠️ 新手使用建议
1. **配合 Composer 使用**：如果在 Cursor 中，建议在 Composer (Ctrl+I) 中使用此 Prompt。
2. **先确认再应用**：看到 AI 给出的代码修改预览时，先点击 "Reject" 或口头确认没问题后，再点击 "Accept"。
```

## 质量标准

在输出前，验证：
- 生成的 Prompt 不存在自相矛盾的约束。
- Prompt 强制执行了安全的开发行为（阻止随意安装依赖和任意删除文件）。
- 语言简单直白、富有指令性，便于目标大模型解析。
