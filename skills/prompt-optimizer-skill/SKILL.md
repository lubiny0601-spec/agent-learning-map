# Prompt Optimizer Skill

## Purpose

This skill guides an Agent in optimizing a beginner's vague or simple natural language prompt into a highly structured, instruction-clear prompt for AI coding assistants (like Cursor, Claude Code, Antigravity, etc.). 

Beginners often write instructions like "帮我做一个博客页面" (Help me build a blog page) or "改一下样式" (Change the styling), which leads to AI hallucinations, broken code, or large, uncontrollable edits. This skill ensures the generated prompt restricts the Agent to a safe, incremental, and highly verified workflow.

Use this skill when the user asks to:
- Generate a prompt template for a specific task.
- Improve a raw prompt they intend to send to another coding assistant.
- Create a system prompt or rules file (e.g., `.cursorrules` or `AGENTS.md`).

## Input Requirements

Gather as much of the following context as possible. If details are missing, use standard industry assumptions (e.g., vanilla JS/CSS for frontend, python for scripting) and mark placeholders as `[待填写]`:

- **Raw Request**: What the user wants to achieve (e.g., "add login function").
- **Target AI Tool**: Cursor Composer, Claude Code, Antigravity, etc.
- **Tech Stack**: HTML, CSS, React, Node.js, Python, etc.
- **Project Context**: Is it a new project or an existing codebase?
- **Constraints**: Design specifications, performance requirements, or library limitations.

## Writing Principles (The Structured Prompt Framework)

The optimized prompt generated by this skill must follow a simplified **CO-STAR** structure adapted for software engineering:

1. **Role & Context (角色与上下文)**: Define the AI's persona (e.g., "You are an expert React developer") and explain the project's background.
2. **Objective (明确任务)**: Break down the raw task into 3-5 concrete, sequential steps. Avoid vague goals.
3. **Tech Stack & Scope (技术栈与范围)**: List exact technologies to use and files that are allowed to be modified.
4. **Defensive Rules (防御性指令 - CRITICAL)**: Add strict guardrails to prevent AI from breaking things:
   - "Do not rewrite entire files; make targeted edits only."
   - "Keep all existing comments, docstrings, and unrelated styles intact."
   - "Ask for my confirmation before installing any new packages or deleting files."
   - "If an error occurs, analyze it step-by-step instead of guessing solutions."
5. **Output Format (输出格式)**: Tell the AI how to respond (e.g., "List modified files first, then explain changes, then provide verification commands").

## Output Structure

Use this structure unless the user requests another format:

```markdown
# Prompt 优化方案

这是为您优化后的结构化 Prompt。你可以直接复制框内的内容发送给您的 AI 编程助手。

## 💡 为什么这样优化
简要说明相比原 Prompt，这个版本做了哪些关键改进（如：限制了修改范围、增加了安全防护、明确了技术栈）。

---

## 📋 复制下方内容发送给 AI

\```text
【角色定义】
你是一个优秀的 [技术栈，如：HTML/JS] 专家。请协助我完成以下开发任务。

【当前上下文】
- 项目类型：[如：静态网页原型]
- 目标文件范围：仅限修改 [文件路径/名称，如：prototype/index.html]，禁止擅自修改其他目录的文件。

【核心任务】
请逐步执行以下操作：
1. [步骤 1，如：在导航栏右侧增加一个“生成器”按钮]
2. [步骤 2，如：为该按钮绑定点击事件，弹出一个简单的模态窗口]
3. [步骤 3，如：在模态窗口中包含三个输入框和一个复制按钮]

【技术要求与约束】
- 使用 Vanilla CSS 进行样式设计，禁止引入第三方 CSS 框架（除非我明确要求）。
- 保持现有的设计美学（如 HSL 色彩配置、圆角过渡等），确保风格一致。
- 确保页面在移动端（响应式）的表现依然良好。

【🛡️ 防御性安全规则】
1. **禁止大范围重写**：只对需要修改的 HTML 代码块进行局部替换，保留所有未涉及的结构和注释。
2. **确认后再操作**：在修改任何文件之前，先向我口头确认你的修改方案和将要修改的文件路径。
3. **禁止擅自下载**：不要运行任何下载依赖或执行外部脚本的命令。

【输出要求】
修改完成后，请告诉我：
1. 你具体修改了哪些行（提供 Diff 或对比说明）。
2. 在浏览器中如何测试该功能是否正常工作。
\```

---

## 🛠️ 新手使用建议
1. **配合 Composer 使用**：如果在 Cursor 中，建议在 Composer (Ctrl+I) 中使用此 Prompt。
2. **先确认再应用**：看到 AI 给出的代码修改预览时，先点击 "Reject" 或口头确认没问题后，再点击 "Accept"。
```

## Quality Standards

Before finalizing, verify:
- The generated prompt does not contain contradictory constraints.
- The prompt enforces safe behaviors (prevents package installation and arbitrary deletion).
- The language is simple, directive, and easy for the target LLM to parse.
