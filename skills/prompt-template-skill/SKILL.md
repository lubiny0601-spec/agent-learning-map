# Prompt 模板生成助手 (Prompt Template Skill)

## 目标

此 Skill 用于帮助初学者将日常繁琐的业务或学术需求，转化为一套高度可复用的、结构严谨的高质量 Prompt（提示词）模板，供 `Agent 学习地图` 项目使用。

初学者往往只擅长向 AI 提问简短的一两句话（如“帮我写一封感谢信”），但这无法充分发挥大模型的真正实力。此 Skill 的目的在于自动为用户生成包含角色定位、上下文信息、明确目标、技术栈约束及防御性安全红线的**开箱即用 Prompt 模板**，并配以大白话说明书，教他们如何修改占位符。

在用户请求进行以下操作时调用此 Skill：
- 为特定医药学术汇报生成 Prompt 模板。
- 为批量处理表格或制作幻灯片大纲生成提示词卡片。
- 为与 AI 模拟医生拜访对练生成配置提示词。
- 寻求将某个特定的日常任务“模板化”。

## 模板设计核心法则 (CO-STAR 法则)

所生成的 Prompt 模板必须包含以下模块结构，并使用清晰的分割线进行划分：

1. **【角色定位】 (Role)**：赋予 AI 合适的专家身份（如：“你是一个医药临床学术专家”）。
2. **【任务上下文】 (Context)**：描述所处场景、受众对象以及所要解决的核心痛点。
3. **【核心目标】 (Objective)**：将任务拆解为具体的、按顺序执行的 3-4 个细分步骤，不使用含糊词汇。
4. **【输入变量】 (Variables)**：用 `[ ]` 标出需要用户填写的具体变量，并教他们如何替换（如：`[请输入具体药品名称]`）。
5. **【🛡️ 安全与合规防御红线】 (Rules)**：加入防止 AI 产生学术幻觉、避免数据泄露的安全指令：
   - “绝对禁止自行捏造文献或指南出处，如果无明确依据，必须明确说明。”
   - “严禁在回答中涉及任何超出说明书范围的推广暗示。”
6. **【输出格式】 (Format)**：限定 AI 回复的版面布局（如使用表格、清单、或者指定字数等）。

## 输出结构

除非用户另有要求，否则请使用以下结构：

```markdown
# 推荐 Prompt 模板：[模板名称]

## 💡 为什么需要这个模板

用大白话解释这个模板解决了什么日常办公痛点，相比简单提问好在哪里。

## 📋 复制下方内容发送给 AI

```text
【角色定义】
你是一个专业的 [角色，如：医药合规顾问]。请协助我完成以下任务。

【背景上下文】
- 受众群体：[如：临床科室主任]
- 任务环境：[如：准备科室学术交流会]

【核心任务】
请逐步执行以下操作：
1. [步骤 1，如：分析输入文献的核心结论]
2. [步骤 2，如：整理出 3 个最适合用于临床说服的学术论点]

【输入内容配置】
以下为我提供的内容，请将变量部分替换为您要分析的数据：
- 目标药品：[ 请输入药品英文缩写或通用名 ]
- 指南名称：[ 请输入具体的国家指南，例如：2024版抗感染指南 ]

【🛡️ 合规与安全红线】
1. **真实性防线**：所有引用的指南页码及结论必须真实，禁止编造学术出处。
2. **推销红线**：严禁出现任何违规或超适应症的疗效断言。

【输出样式】
请使用【Markdown 列表形式】进行结构化输出，保证字数在 500 字以内。
```

---

## 🛠️ 新手使用指南

1. **第一步：复制模板**：一键复制上方虚线框内的全部内容。
2. **第二步：填空修改**：将 `[ ]` 内的中文字，替换为您真实的药品名、指南名或需要处理的文献内容。
3. **第三步：发送运行**：粘贴到 Gemini/ChatGPT 对话框中，点击发送，即可获得稳定高水准的回答。
```

## 质量标准

在输出前，验证：
- 所有的占位符均使用 `[ ]` 标出，且注释清晰易懂。
- 防御性指令与业务背景是完全契合的，没有生硬套用。
- 输出样式限制具体，易于大模型遵循。
- 绝不使用晦涩的开发语言，全篇通俗近人。
